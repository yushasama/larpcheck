Subaru's Piezoelectric Staircase
Statement

In an alternate timeline, Subaru got isekai'd into modern Japan. Japan has installed piezoelectric tiles on every staircase, each tile generates electricity when stepped on. The staircases form a tree of n checkpoints rooted at checkpoint 1, where tile i generates pi units of electricity.

Subaru keeps dying and respawning at different checkpoints. Each time he spawns at u, he wants to know: if he runs through every checkpoint in his subtree, how much total electricity does he generate?

But every time Subaru dies, the mysterious figure also sabotages one tile, changing its power output.

Given q operations, answer all type 2 queries.

1 u x: set tile u to output x.
2 u: query total electricity in subtree of u.

Input

The first line contains a single integer t (1 ≤ t ≤ 104), the number of test cases.

Each test case begins with two integers n and q. The next line contains n integers p1, p2, ..., pn, the initial tile outputs. The next n-1 lines each contain two integers u and v describing an edge. The next q lines each contain an operation of type 1 or 2 as described above.

Output

For each type 2 query, print the total electricity generated in the subtree of u.

Constraints
1 ≤ t ≤ 104
1 ≤ n, q ≤ 2×105
1 ≤ pi, x ≤ 109
Nodes are numbered 1 to n (1-indexed). Edges are undirected.
1 ≤ u, v ≤ n for each edge, u ≠ v
For type 1: 1 ≤ u ≤ n, 1 ≤ x ≤ 109. For type 2: 1 ≤ u ≤ n.
Sum of n and sum of q across all test cases ≤ 2×105
Sample I/O
Sample 1
Input
1 5 4 1 2 3 4 5 1 2 1 3 2 4 2 5 2 2 1 4 10 2 2 2 1
Output
11 17 21
Initial subtree of node 2 = {2,4,5} = 2+4+5 = 11. After tile 4 becomes 10: subtree 2 = 2+10+5 = 17. Subtree 1 = 1+2+3+10+5 = 21.
