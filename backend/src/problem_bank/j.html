Steal a Brainrot
Statement

The Roblox map has n player bases connected by paths forming a tree. Each base i holds a brainrot that prints ri coins per second. You just logged on and you are broke.

You get exactly one stealing run. You pick any base to spawn at, then travel through the tree collecting every brainrot you pass through. You collect every base in the entire connected map, but the order you reach them depends on where you start, and travel along each path costs time.

Your total haul is the sum of ri across all bases you visit, weighted by how early you reach them. Specifically, the value of reaching base i is ri multiplied by (n minus the number of steps it took to reach it from your spawn). Bases you reach later are worth less because their owner logs back on and reclaims them.

You can spawn at any base. Choose the spawn that maximizes your total weighted haul.

For each base u, print the total weighted haul if you spawn there and travel optimally (always moving to unvisited neighbors first via DFS order).

Input

The first line contains a single integer t (1 ≤ t ≤ 104), the number of test cases.

Each test case begins with a single integer n. The next line contains n integers r1, r2, ..., rn, the coin rates. The next n-1 lines each contain two integers u and v, a path between two bases.

Output

For each test case, print n integers on a single line, the total weighted haul for each spawn base from 1 to n.

Constraints
1 ≤ t ≤ 104
1 ≤ n ≤ 2×105
1 ≤ ri ≤ 109
Bases are numbered 1 to n (1-indexed). Paths are undirected.
1 ≤ u, v ≤ n for each path, u ≠ v. No self-loops.
Sum of n across all test cases ≤ 2×105
Sample I/O
Sample 1
Input
1 4 3 1 2 4 1 2 1 3 3 4
Output
24 18 26 20
