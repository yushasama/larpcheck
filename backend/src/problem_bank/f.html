The Larp Circle
Hard Graph Construction
Statement

n people showed up to an anime convention all larping as die-hard fans. Some were deep in their Subahibi lore (watched a YouTube summary). Some swore they read the Chainsaw Man manga (they only know Pochita from TikTok). The Re:Zero group argued about whether Subaru is sigma (none got past episode 3). The JJK crowd was the worst, Hakari mains blasting Tuca Donka, Ryu fans spamming Jane Juliet edits and that panel where he says "lets larp", Gojo believers on Blue Judas, Yuji guys crying on Red Judas. None of them read a single chapter.

To keep up appearances they decided to do larp circles. There are at least 2 people in each circle and each person has exactly two neighbors. If there are 2 people in a circle they share the same neighbor on both sides.

Each person only remembered exactly one neighbor from their circle. Given who each person remembered, determine the minimum and maximum number of larp circles that could have formed.

Input

The first line contains a single integer t (1 ≤ t ≤ 104), the number of test cases.

Each test case begins with a single integer n. The next line contains n integers a1, a2, ..., an, where ai is the one neighbor person i remembered.

Output

For each test case, print two integers on a single line: the minimum and maximum number of larp circles that could have formed.

Constraints
1 ≤ t ≤ 104
2 ≤ n ≤ 2×105
People are numbered 1 to n (1-indexed)
1 ≤ ai ≤ n, ai ≠ i (no one remembered themselves)
Sum of n across all test cases ≤ 2×105
Input is guaranteed to correspond to at least one valid division into larp circles.
Sample I/O
Sample 1
Input
3 6 2 1 4 3 6 5 6 2 3 1 5 6 4 2 2 1
Output
1 3 2 2 1 1
First case min=1: one big circle 1-2-3-4-5-6-1.
First case max=3: (1-2-1), (3-4-3), (5-6-5).
The Subahibi larpers refused to share a circle with the JJK-only-knows-Gojo crowd. Hence max 3.
