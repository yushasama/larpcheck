The M3 Down Payment
Statement

You just raised a seed round for your AI study slop tool. The pitch was simple: "ChatGPT wrapper with a waitlist." Somehow it worked.

You have n investor checks sitting in your business account. You need exactly enough to cover the down payment on an M3 (company car, obviously). The amount you need is target dollars.

Find two checks whose amounts add up to exactly target and return their indices. You may not use the same check twice. There is guaranteed to be exactly one valid answer.

The networking bottle service is already expensed separately. Don't worry about it.

Input

The first line contains a single integer t (1 ≤ t ≤ 104), the number of test cases.

Each test case begins with two integers n and target. The next line contains n integers, the check amounts (-109 ≤ checks[i] ≤ 109).

Output

For each test case, print two integers: the 0-indexed positions of the two checks that sum to target. Print the smaller index first.

Constraints
1 ≤ t ≤ 104
2 ≤ n ≤ 104
-109 ≤ checks[i] ≤ 109
-109 ≤ target ≤ 109
Checks are 0-indexed. Exactly one valid answer exists per test case.
You may not use the same check twice (i ≠ j).
Sum of n across all test cases ≤ 2×105
Notes
Test 1: checks[0] + checks[1] = 2 + 7 = 9. M3 secured. Test 2: checks[1] + checks[2] = 2 + 4 = 6. Bottle service incoming.
Sample I/O
Sample 1
Input
2 4 9 2 7 11 15 3 6 3 2 4
Output
0 1 1 2
